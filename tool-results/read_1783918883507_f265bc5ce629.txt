    62в†’    62в†’## The Animation Decision Framework
    63в†’    63в†’
    64в†’    64в†’Before writing any animation code, answer these questions in order:
    65в†’    65в†’
    66в†’    66в†’### 1. Should this animate at all?
    67в†’    67в†’
    68в†’    68в†’**Ask:** How often will users see this animation?
    69в†’    69в†’
    70в†’    70в†’| Frequency                                                   | Decision                     |
    71в†’    71в†’| ----------------------------------------------------------- | ---------------------------- |
    72в†’    72в†’| 100+ times/day (keyboard shortcuts, command palette toggle) | No animation. Ever.          |
    73в†’    73в†’| Tens of times/day (hover effects, list navigation)          | Remove or drastically reduce |
    74в†’    74в†’| Occasional (modals, drawers, toasts)                        | Standard animation           |
    75в†’    75в†’| Rare/first-time (onboarding, feedback forms, celebrations)  | Can add delight              |
    76в†’    76в†’
    77в†’    77в†’**Never animate keyboard-initiated actions.** These actions are repeated hundreds of times daily. Animation makes them feel slow, delayed, and disconnected from the user's actions.
    78в†’    78в†’
    79в†’    79в†’Raycast has no open/close animation. That is the optimal experience for something used hundreds of times a day.
    80в†’    80в†’
    81в†’    81в†’### 2. What is the purpose?
    82в†’    82в†’
    83в†’    83в†’Every animation must have a clear answer to "why does this animate?"
    84в†’    84в†’
    85в†’    85в†’Valid purposes:
    86в†’    86в†’
    87в†’    87в†’- **Spatial consistency**: toast enters and exits from the same direction, making swipe-to-dismiss feel intuitive
    88в†’    88в†’- **State indication**: a morphing feedback button shows the state change
    89в†’    89в†’- **Explanation**: a marketing animation that shows how a feature works
    90в†’    90в†’- **Feedback**: a button scales down on press, confirming the interface heard the user
    91в†’    91в†’- **Preventing jarring changes**: elements appearing or disappearing without transition feel broken
    92в†’    92в†’
    93в†’    93в†’If the purpose is just "it looks cool" and the user will see it often, don't animate.
    94в†’    94в†’
    95в†’    95в†’### 3. What easing should it use?
    96в†’    96в†’
    97в†’    97в†’Is the element entering or exiting?
    98в†’    98в†’  Yes в†’ ease-out (starts fast, feels responsive)
    99в†’    99в†’  No в†’
   100в†’   100в†’    Is it moving/morphing on screen?
   101в†’   101в†’      Yes в†’ ease-in-out (natural acceleration/deceleration)
   102в†’   102в†’    Is it a hover/color change?
   103в†’   103в†’      Yes в†’ ease
   104в†’   104в†’    Is it constant motion (marquee, progress bar)?
   105в†’   105в†’      Yes в†’ linear
   106в†’   106в†’    Default в†’ ease-out
   107в†’   107в†’
   108в†’   108в†’**Critical: use custom easing curves.** The built-in CSS easings are too weak. They lack the punch that makes animations feel intentional.
   109в†’   109в†’
   110в†’   110в†’```css
   111в†’   111в†’/* Strong ease-out for UI interactions */
   112в†’   112в†’--ease-out: cubic-bezier(0.23, 1, 0.32, 1);
   113в†’   113в†’
   114в†’   114в†’/* Strong ease-in-out for on-screen movement */
   115в†’   115в†’--ease-in-out: cubic-bezier(0.77, 0, 0.175, 1);
   116в†’   116в†’
   117в†’   117в†’/* iOS-like drawer curve (from Ionic Framework) */
   118в†’   118в†’--ease-drawer: cubic-bezier(0.32, 0.72, 0, 1);
   119в†’   119в†’```
   120в†’   120в†’
   121в†’   121в†’**Never use ease-in for UI animations.** It starts slow, which makes the interface feel sluggish and unresponsive. A dropdown with `ease-in` at 300ms _feels_ slower than `ease-out` at the same 300ms, because ease-in delays the initial movement вЂ” the exact moment the user is watching most closely.
   122в†’   122в†’
   123в†’   123в†’**Easing curve resources:** Don't create curves from scratch. Use [easing.dev](https://easing.dev/) or [easings.co](https://easings.co/) to find stronger custom variants of standard easings.
   124в†’   124в†’
   125в†’   125в†’### 4. How fast should it be?
   126в†’   126в†’
   127в†’   127в†’| Element                  | Duration      |
   128в†’   128в†’| ------------------------ | ------------- |
   129в†’   129в†’| Button press feedback    | 100-160ms     |
   130в†’   130в†’| Tooltips, small popovers | 125-200ms     |
   131в†’   131в†’| Dropdowns, selects       | 150-250ms     |
   132в†’   132в†’| Modals, drawers          | 200-500ms     |
   133в†’   133в†’| Marketing/explanatory    | Can be longer |
   134в†’   134в†’
   135в†’   135в†’**Rule: UI animations should stay under 300ms.** A 180ms dropdown feels more responsive than a 400ms one. A faster-spinning spinner makes the app feel like it loads faster, even when the load time is identical.
   136в†’   136в†’
   137в†’   137в†’### Perceived performance
   138в†’   138в†’
   139в†’   139в†’Speed in animation is not just about feeling snappy вЂ” it directly affects how users perceive your app's performance:
   140в†’   140в†’
   141в†’   141в†’- A **fast-spinning spinner** makes loading feel faster (same load time, different perception)
   142в†’   142в†’- A **180ms select** animation feels more responsive than a **400ms** one
   143в†’   143в†’- **Instant tooltips** after the first one is open (skip delay + skip animation) make the whole toolbar feel faster
   144в†’   144в†’
   145в†’   145в†’The perception of speed matters as much as actual speed. Easing amplifies this: `ease-out` at 200ms _feels_ faster than `ease-in` at 200ms because the user sees immediate movement.
   146в†’   146в†’
   147в†’   147в†’## Spring Animations
   148в†’   148в†’
   149в†’   149в†’Springs feel more natural than duration-based animations because they simulate real physics. They don't have fixed durations вЂ” they settle based on physical parameters.
   150в†’   150в†’
   151в†’   151в†’### When to use springs
   152в†’   152в†’
   153в†’   153в†’- Drag interactions with momentum
   154в†’   154в†’- Elements that should feel "alive" (like Apple's Dynamic Island)
   155в†’   155в†’- Gestures that can be interrupted mid-animation
   156в†’   156в†’- Decorative mouse-tracking interactions
   157в†’   157в†’
   158в†’   158в†’### Spring-based mouse interactions
   159в†’   159в†’
   160в†’   160в†’Tying visual changes directly to mouse position feels artificial because it lacks motion. Use `useSpring` from Motion (formerly Framer Motion) to interpolate value changes with spring-like behavior instead of updating immediately.
   161в†’   161в†’
   162в†’   162в†’```jsx
   163в†’   163в†’import { useSpring } from 'framer-motion';
   164в†’   164в†’
   165в†’   165в†’// Without spring: feels artificial, instant
   166в†’   166в†’const rotation = mouseX * 0.1;
   167в†’   167в†’
   168в†’   168в†’// With spring: feels natural, has momentum
   169в†’   169в†’const springRotation = useSpring(mouseX * 0.1, {
   170в†’   170в†’  stiffness: 100,
   171в†’   171в†’  damping: 10,
   172в†’   172в†’});
   173в†’   173в†’```
   174в†’   174в†’
   175в†’   175в†’This works because the animation is **decorative** вЂ” it doesn't serve a function. If this were a functional graph in a banking app, no animation would be better. Know when decoration helps and when it hinders.
   176в†’   176в†’
   177в†’   177в†’### Spring configuration
   178в†’   178в†’
   179в†’   179в†’**Apple's approach (recommended вЂ” easier to reason about):**
   180в†’   180в†’
   181в†’   181в†’```js
   182в†’   182в†’{ type: "spring", duration: 0.5, bounce: 0.2 }
   183в†’   183в†’```
   184в†’   184в†’
   185в†’   185в†’**Traditional physics (more control):**
   186в†’   186в†’
   187в†’   187в†’```js
   188в†’   188в†’{ type: "spring", mass: 1, stiffness: 100, damping: 10 }
   189в†’   189в†’```
   190в†’   190в†’
   191в†’   191в†’Keep bounce subtle (0.1-0.3) when used. Avoid bounce in most UI contexts. Use it for drag-to-dismiss and playful interactions.
   192в†’   192в†’
   193в†’   193в†’### Interruptibility advantage
   194в†’   194в†’
   195в†’   195в†’Springs maintain velocity when interrupted вЂ” CSS animations and keyframes restart from zero. This makes springs ideal for gestures users might change mid-motion. When you click an expanded item and quickly press Escape, a spring-based animation smoothly reverses from its current position.
   196в†’   196в†’
   197в†’   197в†’## Component Building Principles
   198в†’   198в†’
   199в†’   199в†’### Buttons must feel responsive
   200в†’   200в†’
   201в†’   201в†’Add `transform: scale(0.97)` on `:active`. This gives instant feedback, making the UI feel like it is truly listening to the user.
   202в†’   202в†’
   203в†’   203в†’```css
   204в†’   204в†’.button {
   205в†’   205в†’  transition: transform 160ms ease-out;
   206в†’   206в†’}
   207в†’   207в†’
   208в†’   208в†’.button:active {
   209в†’   209в†’  transform: scale(0.97);
   210в†’   210в†’}
   211в†’   211в†’```
   212в†’   212в†’
   213в†’   213в†’This applies to any pressable element. The scale should be subtle (0.95-0.98).
   214в†’   214в†’
   215в†’   215в†’### Never animate from scale(0)
   216в†’   216в†’
   217в†’   217в†’Nothing in the real world disappears and reappears completely. Elements animating from `scale(0)` look like they come out of nowhere.
   218в†’   218в†’
   219в†’   219в†’Start from `scale(0.9)` or higher, combined with opacity. Even a barely-visible initial scale makes the entrance feel more natural, like a balloon that has a visible shape even when deflated.
   220в†’   220в†’
   221в†’   221в†’```css
   222в†’   222в†’/* Bad */
   223в†’   223в†’.entering {
   224в†’   224в†’  transform: scale(0);
   225в†’   225в†’}
   226в†’   226в†’
   227в†’   227в†’/* Good */
   228в†’   228в†’.entering {
   229в†’   229в†’  transform: scale(0.95);
   230в†’   230в†’  opacity: 0;
   231в†’   231в†’}
   232в†’   232в†’```
   233в†’   233в†’
   234в†’   234в†’### Make popovers origin-aware
   235в†’   235в†’
   236в†’   236в†’Popovers should scale in from their trigger, not from center. The default `transform-origin: center` is wrong for almost every popover. **Exception: modals.** Modals should keep `transform-origin: center` because they are not anchored to a specific trigger вЂ” they appear centered in the viewport.
   237в†’   237в†’
   238в†’   238в†’```css
   239в†’   239в†’/* Radix UI */
   240в†’   240в†’.popover {
   241в†’   241в†’  transform-origin: var(--radix-popover-content-transform-origin);
   242в†’   242в†’}
   243в†’   243в†’
   244в†’   244в†’/* Base UI */
   245в†’   245в†’.popover {
   246в†’   246в†’  transform-origin: var(--transform-origin);
   247в†’   247в†’}
   248в†’   248в†’```
   249в†’   249в†’
   250в†’   250в†’Whether the user notices the difference individually does not matter. In the aggregate, unseen details become visible. They compound.
   251в†’   251в†’
   252в†’   252в†’### Tooltips: skip delay on subsequent hovers
   253в†’   253в†’
   254в†’   254в†’Tooltips should delay before appearing to prevent accidental activation. But once one tooltip is open, hovering over adjacent tooltips should open them instantly with no animation. This feels faster without defeating the purpose of the initial delay.
   255в†’   255в†’
   256в†’   256в†’```css
   257в†’   257в†’.tooltip {
   258в†’   258в†’  transition: transform 125ms ease-out, opacity 125ms ease-out;
   259в†’   259в†’  transform-origin: var(--transform-origin);
   260в†’   260в†’}
   261в†’   261в†’
   262в†’   262в†’.tooltip[data-starting-style],
   263в†’   263в†’.tooltip[data-ending-style] {
   264в†’   264в†’  opacity: 0;
   265в†’   265в†’  transform: scale(0.97);
   266в†’   266в†’}
   267в†’   267в†’
   268в†’   268в†’/* Skip animation on subsequent tooltips */
   269в†’   269в†’.tooltip[data-instant] {
   270в†’   270в†’  transition-duration: 0ms;
   271в†’   271в†’}
   272в†’   272в†’```
   273в†’   273в†’
   274в†’   274в†’### Use CSS transitions over keyframes for interruptible UI
   275в†’   275в†’
   276в†’   276в†’CSS transitions can be interrupted and retargeted mid-animation. Keyframes restart from zero. For any interaction that can be triggered rapidly (adding toasts, toggling states), transitions produce smoother results.
   277в†’   277в†’
   278в†’   278в†’```css
   279в†’   279в†’/* Interruptible - good for UI */
   280в†’   280в†’.toast {
   281в†’   281в†’  transition: transform 400ms ease;
   282в†’   282в†’}
   283в†’   283в†’
   284в†’   284в†’/* Not interruptible - avoid for dynamic UI */
   285в†’   285в†’@keyframes slideIn {
   286в†’   286в†’  from {
   287в†’   287в†’    transform: translateY(100%);
   288в†’   288в†’  }
   289в†’   289в†’  to {
   290в†’   290в†’    transform: translateY(0);
   291в†’   291в†’  }
   292в†’   292в†’}
   293в†’   293в†’```
   294в†’   294в†’
   295в†’   295в†’### Use blur to mask imperfect transitions
   296в†’   296в†’
   297в†’   297в†’When a crossfade between two states feels off despite trying different easings and durations, add subtle `filter: blur(2px)` during the transition.
   298в†’   298в†’
   299в†’   299в†’**Why blur works:** Without blur, you see two distinct objects during a crossfade вЂ” the old state and the new state overlapping. This looks unnatural. Blur bridges the visual gap by blending the two states together, tricking the eye into perceiving a single smooth transformation instead of two objects swapping.
   300в†’   300в†’
   301в†’   301в†’Combine blur with scale-on-press (`scale(0.97)`) for a polished button state transition:
   302в†’   302в†’
   303в†’   303в†’```css
   304в†’   304в†’.button {
   305в†’   305в†’  transition: transform 160ms ease-out;
   306в†’   306в†’}
   307в†’   307в†’
   308в†’   308в†’.button:active {
   309в†’   309в†’  transform: scale(0.97);
   310в†’   310в†’}
   311в†’   311в†’
   312в†’   312в†’.button-content {
   313в†’   313в†’  transition: filter 200ms ease, opacity 200ms ease;
   314в†’   314в†’}
   315в†’   315в†’
   316в†’   316в†’.button-content.transitioning {
   317в†’   317в†’  filter: blur(2px);
   318в†’   318в†’  opacity: 0.7;
   319в†’   319в†’}
   320в†’   320в†’```
   321в†’   321в†’
   322в†’   322в†’Keep blur under 20px. Heavy blur is expensive, especially in Safari.
   323в†’   323в†’
   324в†’   324в†’### Animate enter states with @starting-style
   325в†’   325в†’
   326в†’   326в†’The modern CSS way to animate element entry without JavaScript:
   327в†’   327в†’
   328в†’   328в†’```css
   329в†’   329в†’.toast {
   330в†’   330в†’  opacity: 1;
   331в†’   331в†’  transform: translateY(0);
   332в†’   332в†’  transition: opacity 400ms ease, transform 400ms ease;
   333в†’   333в†’
   334в†’   334в†’  @starting-style {
   335в†’   335в†’    opacity: 0;
   336в†’   336в†’    transform: translateY(100%);
   337в†’   337в†’  }
   338в†’   338в†’}
   339в†’   339в†’```
   340в†’   340в†’
   341в†’   341в†’This replaces the common React pattern of using `useEffect` to set `mounted: true` after initial render. Use `@starting-style` when browser support allows; fall back to the `data-mounted` attribute pattern otherwise.
   342в†’   342в†’
   343в†’   343в†’```jsx
   344в†’   344в†’// Legacy pattern (still works everywhere)
   345в†’   345в†’useEffect(() => {
   346в†’   346в†’  setMounted(true);
   347в†’   347в†’}, []);
   348в†’   348в†’// <div data-mounted={mounted}>
   349в†’   349в†’```

   351в†’   351в†’## CSS Transform Mastery
   352в†’   352в†’
   353в†’   353в†’### translateY with percentages
   354в†’   354в†’
   355в†’   355в†’Percentage values in `translate()` are relative to the element's own size. Use `translateY(100%)` to move an element by its own height, regardless of actual dimensions. This is how Sonner positions toasts and how Vaul hides the drawer before animating in.
   356в†’   356в†’
   357в†’   357в†’```css
   358в†’   358в†’/* Works regardless of drawer height */
   359в†’   359в†’.drawer-hidden {
   360в†’   360в†’  transform: translateY(100%);
   361в†’   361в†’}
   362в†’   362в†’
   363в†’   363в†’/* Works regardless of toast height */
   364в†’   364в†’.toast-enter {
   365в†’   365в†’  transform: translateY(-100%);
   366в†’   366в†’}
   367в†’   367в†’```
   368в†’   368в†’
   369в†’   369в†’Prefer percentages over hardcoded pixel values. They are less error-prone and adapt to content.
   370в†’   370в†’
   371в†’   371в†’### scale() scales children too
   372в†’   372в†’
   373в†’   373в†’Unlike `width`/`height`, `scale()` also scales an element's children. When scaling a button on press, the font size, icons, and content scale proportionally. This is a feature, not a bug.
   374в†’   374в†’
   375в†’   375в†’### 3D transforms for depth
   376в†’   376в†’
   377в†’   377в†’`rotateX()`, `rotateY()` with `transform-style: preserve-3d` create real 3D effects in CSS. Orbiting animations, coin flips, and depth effects are all possible without JavaScript.
   378в†’   378в†’
   379в†’   379в†’```css
   380в†’   380в†’.wrapper {
   381в†’   381в†’  transform-style: preserve-3d;
   382в†’   382в†’}
   383в†’   383в†’
   384в†’   384в†’@keyframes orbit {
   385в†’   385в†’  from {
   386в†’   386в†’    transform: translate(-50%, -50%) rotateY(0deg) translateZ(72px) rotateY(360deg);
   387в†’   387в†’  }
   388в†’   388в†’  to {
   389в†’   389в†’    transform: translate(-50%, -50%) rotateY(360deg) translateZ(72px) rotateY(0deg);
   390в†’   390в†’  }
   391в†’   391в†’}
   392в†’   392в†’```
   393в†’   393в†’
   394в†’   394в†’### transform-origin
   395в†’   395в†’
   396в†’   396в†’Every element has an anchor point from which transforms execute. The default is center. Set it to match where the trigger lives for origin-aware interactions.
   397в†’   397в†’
   398в†’   398в†’## clip-path for Animation
   399в†’   399в†’
   400в†’   400в†’`clip-path` is not just for shapes. It is one of the most powerful animation tools in CSS.
   401в†’   401в†’
   402в†’   402в†’### The inset shape
   403в†’   403в†’
   404в†’   404в†’`clip-path: inset(top right bottom left)` defines a rectangular clipping region. Each value "eats" into the element from that side.
   405в†’   405в†’
   406в†’   406в†’```css
   407в†’   407в†’/* Fully hidden from right */
   408в†’   408в†’.hidden {
   409в†’   409в†’  clip-path: inset(0 100% 0 0);
   410в†’   410в†’}
   411в†’   411в†’
   412в†’   412в†’/* Fully visible */
   413в†’   413в†’.visible {
   414в†’   414в†’  clip-path: inset(0 0 0 0);
   415в†’   415в†’}
   416в†’   416в†’
   417в†’   417в†’/* Reveal from left to right */
   418в†’   418в†’.overlay {
   419в†’   419в†’  clip-path: inset(0 100% 0 0);
   420в†’   420в†’  transition: clip-path 200ms ease-out;
   421в†’   421в†’}
   422в†’   422в†’.button:active .overlay {
   423в†’   423в†’  clip-path: inset(0 0 0 0);
   424в†’   424в†’  transition: clip-path 2s linear;
   425в†’   425в†’}
   426в†’   426в†’```
   427в†’   427в†’
   428в†’   428в†’### Tabs with perfect color transitions
   429в†’   429в†’
   430в†’   430в†’Duplicate the tab list. Style the copy as "active" (different background, different text color). Clip the copy so only the active tab is visible. Animate the clip on tab change. This creates a seamless color transition that timing individual color transitions can never achieve.
   431в†’   431в†’
   432в†’   432в†’### Hold-to-delete pattern
   433в†’   433в†’
   434в†’   434в†’Use `clip-path: inset(0 100% 0 0)` on a colored overlay. On `:active`, transition to `inset(0 0 0 0)` over 2s with linear timing. On release, snap back with 200ms ease-out. Add `scale(0.97)` on the button for press feedback.
   435в†’   435в†’
   436в†’   436в†’### Image reveals on scroll
   437в†’   437в†’
   438в†’   438в†’Start with `clip-path: inset(0 0 100% 0)` (hidden from bottom). Animate to `inset(0 0 0 0)` when the element enters the viewport. Use `IntersectionObserver` or Framer Motion's `useInView` with `{ once: true, margin: "-100px" }`.
   439в†’   439в†’
   440в†’   440в†’### Comparison sliders
   441в†’   441в†’
   442в†’   442в†’Overlay two images. Clip the top one with `clip-path: inset(0 50% 0 0)`. Adjust the right inset value based on drag position. No extra DOM elements needed, fully hardware-accelerated.
   443в†’   443в†’
   444в†’   444в†’## Gesture and Drag Interactions
   445в†’   445в†’
   446в†’   446в†’### Momentum-based dismissal
   447в†’   447в†’
   448в†’   448в†’Don't require dragging past a threshold. Calculate velocity: `Math.abs(dragDistance) / elapsedTime`. If velocity exceeds ~0.11, dismiss regardless of distance. A quick flick should be enough.
   449в†’   449в†’
   450в†’   450в†’```js
   451в†’   451в†’const timeTaken = new Date().getTime() - dragStartTime.current.getTime();
   452в†’   452в†’const velocity = Math.abs(swipeAmount) / timeTaken;
   453в†’   453в†’
   454в†’   454в†’if (Math.abs(swipeAmount) >= SWIPE_THRESHOLD || velocity > 0.11) {
   455в†’   455в†’  dismiss();
   456в†’   456в†’}
   457в†’   457в†’```
   458в†’   458в†’
   459в†’   459в†’### Damping at boundaries
   460в†’   460в†’
   461в†’   461в†’When a user drags past the natural boundary (e.g., dragging a drawer up when already at top), apply damping. The more they drag, the less the element moves. Things in real life don't suddenly stop; they slow down first.
   462в†’   462в†’
   463в†’   463в†’### Pointer capture for drag
   464в†’   464в†’
   465в†’   465в†’Once dragging starts, set the element to capture all pointer events. This ensures dragging continues even if the pointer leaves the element bounds.
   466в†’   466в†’
   467в†’   467в†’### Multi-touch protection
   468в†’   468в†’
   469в†’   469в†’Ignore additional touch points after the initial drag begins. Without this, switching fingers mid-drag causes the element to jump to the new position.
   470в†’   470в†’
   471в†’   471в†’```js
   472в†’   472в†’function onPress() {
   473в†’   473в†’  if (isDragging) return;
   474в†’   474в†’  // Start drag...
   475в†’   475в†’}
   476в†’   476в†’```
   477в†’   477в†’
   478в†’   478в†’### Friction instead of hard stops
   479в†’   479в†’
   480в†’   480в†’Instead of preventing upward drag entirely, allow it with increasing friction. It feels more natural than hitting an invisible wall.
   481в†’   481в†’
   482в†’   482в†’## Performance Rules
   483в†’   483в†’
   484в†’   484в†’### Only animate transform and opacity
   485в†’   485в†’
   486в†’   486в†’These properties skip layout and paint, running on the GPU. Animating `padding`, `margin`, `height`, or `width` triggers all three rendering steps.
   487в†’   487в†’
   488в†’   488в†’### CSS variables are inheritable
   489в†’   489в†’
   490в†’   490в†’Changing a CSS variable on a parent recalculates styles for all children. In a drawer with many items, updating `--swipe-amount` on the container causes expensive style recalculation. Update `transform` directly on the element instead.
   491в†’   491в†’
   492в†’   492в†’```js
   493в†’   493в†’// Bad: triggers recalc on all children
   494в†’   494в†’element.style.setProperty('--swipe-amount', `${distance}px`);
   495в†’   495в†’
   496в†’   496в†’// Good: only affects this element
   497в†’   497в†’element.style.transform = `translateY(${distance}px)`;
   498в†’   498в†’```
   499в†’   499в†’
   500в†’   500в†’### Framer Motion hardware acceleration caveat
   501в†’   501в†’
   502в†’   502в†’Framer Motion's shorthand properties (`x`, `y`, `scale`) are NOT hardware-accelerated. They use `requestAnimationFrame` on the main thread. For hardware acceleration, use the full `transform` string:
   503в†’   503в†’
   504в†’   504в†’```jsx
   505в†’   505в†’// NOT hardware accelerated (convenient but drops frames under load)
   506в†’   506в†’<motion.div animate={{ x: 100 }} />
   507в†’   507в†’
   508в†’   508в†’// Hardware accelerated (stays smooth even when main thread is busy)
   509в†’   509в†’<motion.div animate={{ transform: "translateX(100px)" }} />
   510в†’   510в†’```
   511в†’   511в†’
   512в†’   512в†’This matters when the browser is simultaneously loading content, running scripts, or painting. At Vercel, the dashboard tab animation used Shared Layout Animations and dropped frames during page loads. Switching to CSS animations (off main thread) fixed it.
   513в†’   513в†’
   514в†’   514в†’### CSS animations beat JS under load
   515в†’   515в†’
   516в†’   516в†’CSS animations run off the main thread. When the browser is busy loading a new page, Framer Motion animations (using `requestAnimationFrame`) drop frames. CSS animations remain smooth. Use CSS for predetermined animations; JS for dynamic, interruptible ones.
   517в†’   517в†’
   518в†’   518в†’### Use WAAPI for programmatic CSS animations
   519в†’   519в†’
   520в†’   520в†’The Web Animations API gives you JavaScript control with CSS performance. Hardware-accelerated, interruptible, and no library needed.
   521в†’   521в†’
   522в†’   522в†’```js
   523в†’   523в†’element.animate([{ clipPath: 'inset(0 0 100% 0)' }, { clipPath: 'inset(0 0 0 0)' }], {
   524в†’   524в†’  duration: 1000,
   525в†’   525в†’  fill: 'forwards',
   526в†’   526в†’  easing: 'cubic-bezier(0.77, 0, 0.175, 1)',
   527в†’   527в†’});
   528в†’   528в†’```
   529в†’   529в†’
   530в†’   530в†’## Accessibility
   531в†’   531в†’
   532в†’   532в†’### prefers-reduced-motion
   533в†’   533в†’
   534в†’   534в†’Animations can cause motion sickness. Reduced motion means fewer and gentler animations, not zero. Keep opacity and color transitions that aid comprehension. Remove movement and position animations.
   535в†’   535в†’
   536в†’   536в†’```css
   537в†’   537в†’@media (prefers-reduced-motion: reduce) {
   538в†’   538в†’  .element {
   539в†’   539в†’    animation: fade 0.2s ease;
   540в†’   540в†’    /* No transform-based motion */
   541в†’   541в†’  }
   542в†’   542в†’}
   543в†’   543в†’```
   544в†’   544в†’
   545в†’   545в†’```jsx
   546в†’   546в†’const shouldReduceMotion = useReducedMotion();
   547в†’   547в†’const closedX = shouldReduceMotion ? 0 : '-100%';
   548в†’   548в†’```
   549в†’   549в†’
   550в†’   550в†’### Touch device hover states
   551в†’   551в†’
   552в†’   552в†’```css
   553в†’   553в†’@media (hover: hover) and (pointer: fine) {
   554в†’   554в†’  .element:hover {
   555в†’   555в†’    transform: scale(1.05);
   556в†’   556в†’  }
   557в†’   557в†’}
   558в†’   558в†’```
   559в†’   559в†’
   560в†’   560в†’Touch devices trigger hover on tap, causing false positives. Gate hover animations behind this media query.
   561в†’   561в†’
   562в†’   562в†’## The Sonner Principles (Building Loved Components)
   563в†’   563в†’
   564в†’   564в†’These principles come from building Sonner (13M+ weekly npm downloads) and apply to any component:
   565в†’   565в†’
   566в†’   566в†’1. **Developer experience is key.** No hooks, no context, no complex setup. Insert `<Toaster />` once, call `toast()` from anywhere. The less friction to adopt, the more people will use it.
   567в†’   567в†’
   568в†’   568в†’2. **Good defaults matter more than options.** Ship beautiful out of the box. Most users never customize. The default easing, timing, and visual design should be excellent.
   569в†’   569в†’
   570в†’   570в†’3. **Naming creates identity.** "Sonner" (French for "to ring") feels more elegant than "react-toast". Sacrifice discoverability for memorability when appropriate.
   571в†’   571в†’
   572в†’   572в†’4. **Handle edge cases invisibly.** Pause toast timers when the tab is hidden. Fill gaps between stacked toasts with pseudo-elements to maintain hover state. Capture pointer events during drag. Users never notice these, and that is exactly right.
   573в†’   573в†’
   574в†’   574в†’5. **Use transitions, not keyframes, for dynamic UI.** Toasts are added rapidly. Keyframes restart from zero on interruption. Transitions retarget smoothly.
   575в†’   575в†’
   576в†’   576в†’6. **Build a great documentation site.** Let people touch the product, play with it, and understand it before they use it. Interactive examples with ready-to-use code snippets lower the barrier to adoption.
   577в†’   577в†’
   578в†’   578в†’### Cohesion matters
   579в†’   579в†’
   580в†’   580в†’Sonner's animation feels satisfying partly because the whole experience is cohesive. The easing and duration fit the vibe of the library. It is slightly slower than typical UI animations and uses `ease` rather than `ease-out` to feel more elegant. The animation style matches the toast design, the page design, the name вЂ” everything is in harmony.
   581в†’   581в†’
   582в†’   582в†’When choosing animation values, consider the personality of the component. A playful component can be bouncier. A professional dashboard should be crisp and fast. Match the motion to the mood.
   583в†’   583в†’
   584в†’   584в†’### The opacity + height combination
   585в†’   585в†’
   586в†’   586в†’When items enter and exit a list (like Family's drawer), the opacity change must work well with the height animation. This is often trial and error. There is no formula вЂ” you adjust until it feels right.
   587в†’   587в†’
   588в†’   588в†’### Review your work the next day
   589в†’   589в†’
   590в†’   590в†’Review animations with fresh eyes. You notice imperfections the next day that you missed during development. Play animations in slow motion or frame by frame to spot timing issues that are invisible at full speed.
   591в†’   591в†’
   592в†’   592в†’### Asymmetric enter/exit timing
   593в†’   593в†’
   594в†’   594в†’Pressing should be slow when it needs to be deliberate (hold-to-delete: 2s linear), but release should always be snappy (200ms ease-out). This pattern applies broadly: slow where the user is deciding, fast where the system is responding.
   595в†’   595в†’
   596в†’   596в†’```css
   597в†’   597в†’/* Release: fast */
   598в†’   598в†’.overlay {
   599в†’   599в†’  transition: clip-path 200ms ease-out;
   600в†’   600в†’}
   601в†’   601в†’
   602в†’   602в†’/* Press: slow and deliberate */
   603в†’   603в†’.button:active .overlay {
   604в†’   604в†’  transition: clip-path 2s linear;
   605в†’   605в†’}
   606в†’   606в†’```
   607в†’   607в†’
   608в†’   608в†’## Stagger Animations
   609в†’   609в†’
   610в†’   610в†’When multiple elements enter together, stagger their appearance. Each element animates in with a small delay after the previous one. This creates a cascading effect that feels more natural than everything appearing at once.
   611в†’   611в†’
   612в†’   612в†’```css
   613в†’   613в†’.item {
   614в†’   614в†’  opacity: 0;
   615в†’   615в†’  transform: translateY(8px);
   616в†’   616в†’  animation: fadeIn 300ms ease-out forwards;
   617в†’   617в†’}
   618в†’   618в†’
   619в†’   619в†’.item:nth-child(1) {
   620в†’   620в†’  animation-delay: 0ms;
   621в†’   621в†’}
   622в†’   622в†’.item:nth-child(2) {
   623в†’   623в†’  animation-delay: 50ms;
   624в†’   624в†’}
   625в†’   625в†’.item:nth-child(3) {
   626в†’   626в†’  animation-delay: 100ms;
   627в†’   627в†’}
   628в†’   628в†’.item:nth-child(4) {
   629в†’   629в†’  animation-delay: 150ms;
   630в†’   630в†’}
   631в†’   631в†’
   632в†’   632в†’@keyframes fadeIn {
   633в†’   633в†’  to {
   634в†’   634в†’    opacity: 1;
   635в†’   635в†’    transform: translateY(0);
   636в†’   636в†’  }
   637в†’   637в†’}
   638в†’   638в†’```
   639в†’   639в†’
   640в†’   640в†’Keep stagger delays short (30-80ms between items). Long delays make the interface feel slow. Stagger is decorative вЂ” never block interaction while stagger animations are playing.
   641в†’   641в†’
   642в†’   642в†’## Debugging Animations
   643в†’   643в†’
   644в†’   644в†’### Slow motion testing
   645в†’   645в†’
   646в†’   646в†’Play animations at reduced speed to spot issues invisible at full speed. Temporarily increase duration to 2-5x normal, or use browser DevTools animation inspector to slow playback.
   647в†’   647в†’
   648в†’   648в†’Things to look for in slow motion:
   649в†’   649в†’
   650в†’   650в†’- Do colors transition smoothly, or do you see two distinct states overlapping?
   651в†’   651в†’- Does the easing feel right, or does it start/stop abruptly?
   652в†’   652в†’- Is the transform-origin correct, or does the element scale from the wrong point?
   653в†’   653в†’- Are multiple animated properties (opacity, transform, color) in sync?
   654в†’   654в†’
   655в†’   655в†’### Frame-by-frame inspection
   656в†’   656в†’
   657в†’   657в†’Step through animations frame by frame in Chrome DevTools (Animations panel). This reveals timing issues between coordinated properties that you cannot see at full speed.
   658в†’   658в†’
   659в†’   659в†’### Test on real devices
   660в†’   660в†’
   661в†’   661в†’For touch interactions (drawers, swipe gestures), test on physical devices. Connect your phone via USB, visit your local dev server by IP address, and use Safari's remote devtools. The Xcode Simulator is an alternative but real hardware is better for gesture testing.

   663в†’   663в†’## Review Checklist
   664в†’   664в†’
   665в†’   665в†’When reviewing UI code, check for:
   666в†’   666в†’
   667в†’   667в†’| Issue                                      | Fix                                                              |
   668в†’   668в†’| ------------------------------------------ | ---------------------------------------------------------------- |
   669в†’   669в†’| `transition: all`                          | Specify exact properties: `transition: transform 200ms ease-out` |
   670в†’   670в†’| `scale(0)` entry animation                 | Start from `scale(0.95)` with `opacity: 0`                       |
   671в†’   671в†’| `ease-in` on UI element                    | Switch to `ease-out` or custom curve                             |
   672в†’   672в†’| `transform-origin: center` on popover      | Set to trigger location or use Radix/Base UI CSS variable (modals are exempt вЂ” keep centered) |
   673в†’   673в†’| Animation on keyboard action               | Remove animation entirely                                        |
   674в†’   674в†’| Duration > 300ms on UI element             | Reduce to 150-250ms                                              |
   675в†’   675в†’| Hover animation without media query        | Add `@media (hover: hover) and (pointer: fine)`                  |
   676в†’   676в†’| Keyframes on rapidly-triggered element     | Use CSS transitions for interruptibility                         |
   677в†’   677в†’| Framer Motion `x`/`y` props under load     | Use `transform: "translateX()"` for hardware acceleration        |
   678в†’   678в†’| Same enter/exit transition speed           | Make exit faster than enter (e.g., enter 2s, exit 200ms)         |
   679в†’   679в†’| Elements all appear at once                | Add stagger delay (30-80ms between items)                        |
